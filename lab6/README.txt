============================================================

DUNGEON WALKER

============================================================

Аркадный лабиринт в стиле Wolfenstein 3D на WPF
За геймплейную основу был взят Pacman.

============================================================
КОНСТАНТЫ И СЕТКА
============================================================

CellSize  = 40    — размер одной ячейки в пикселях
MapWidth  = 19    — ширина карты в ячейках (760 / 40)
MapHeight = 15    — высота карты в ячейках (600 / 40)

Карта хранится как int[,]:
  0 = проход
  1 = стена (кирпич, уровень 1)
  2 = стена (камень, уровень 2)
  3 = стена (синяя плитка, уровень 3)


============================================================
УРОВНИ
============================================================

В игре 4 типа уровней:

1. Подземелье Замка (фиксированный)
   - Кирпичные стены (wall_brick.png)
   - Коридоры и комнаты
   - Классический лабиринт

2. Темница (фиксированный)
   - Каменные стены (wall_stone.png)
   - Открытое пространство с колоннами
   - Несколько проходов в колоннах

3. Сокровищница (фиксированный)
   - Синяя плитка (wall_blue.png)
   - Узкий коридор-змейка
   - Мало места для манёвра

4. Бесконечное Подземелье (рандомная генерация)
   - Случайная текстура стен
   - Генерация методом Random Walk
   - При каждом запуске/рестарте — новая карта


============================================================
РАНДОМНАЯ ГЕНЕРАЦИЯ (Бесконечное Подземелье)
============================================================

Алгоритм: Random Walk ("крот")

1. Вся карта заполняется стенами (1)
2. Выбирается случайная стартовая клетка
3. "Крот" ходит по карте:
   - Случайное направление (из 4)
   - Шаг на 2 клетки
   - Открывает целевую и промежуточную клетки
4. Цель: открыть ~50% клеток карты
5. Границы всегда остаются стенами
6. Клетка (1,1) принудительно открыта для старта игрока

Перед запуском показывается экран загрузки "СТРОИМ СТЕНЫ..."
с анимированными точками (3 секунды).


============================================================
ИГРОК
============================================================

Позиция: playerGridX, playerGridY (в сетке)
Скорость: moveSpeed = 0.12 сек/клетка
Спрайт: player_attack.png (36x36 пикселей)

Характеристики:
  lives  = 3       — количество жизней
  score  = 0       — собрано сокровищ

Флаги состояния:
  isMoving         — блокировка движения во время анимации
  isInvulnerable   — неуязвимость после получения урона
  isGameRunning    — активна ли игра

Движение (TryMovePlayer):
  1. Проверка: не выходит за границы карты
  2. Проверка: целевая клетка — проход (map[y,x] == 0)
  3. Проверка: не в процессе анимации (isMoving)
  4. Блокировка: isMoving = true
  5. Воспроизведение звука шагов (step.wav)
  6. Запуск двух DoubleAnimation:
     - Canvas.Left: от текущей позиции до новой
     - Canvas.Top: от текущей позиции до новой
     - Duration = moveSpeed
     - FillBehavior = HoldEnd
  7. По завершении анимации:
     - isMoving = false
     - Проверка сбора сокровищ


============================================================
ВРАГИ
============================================================

Класс Enemy:
  GridX, GridY      — позиция в сетке
  PatrolPath         — маршрут патрулирования (список клеток)
  PatrolIndex        — индекс текущей точки маршрута
  IsAlive            — жив ли враг
  IsMoving           — в процессе анимации
  IsChasing          — режим преследования игрока
  BaseSpeed          — скорость патруля
  ChaseSpeed         — скорость преследования (BaseSpeed * 1.6)

Количество врагов зависит от сложности:
  Легкая:   3 врага
  Нормал:   4 врага
  Хард:     4 врага
  Ад:       5 врагов

Спавн (SpawnEnemies):
  1. Поиск всех проходимых клеток вдали от игрока (>3 клеток)
  2. Случайный выбор enemyCount позиций
  3. Генерация маршрута патруля через BFS
  4. Создание спрайта (enemy_move.png, 36x36)
  5. Случайная базовая скорость (учёт difficultyMultiplier)
  6. Запуск движения со случайной задержкой (200-800 мс)


============================================================
BFS — ПОИСК ПУТИ
============================================================

Метод: FindFullPath(startX, startY, endX, endY)

Алгоритм:
  1. Очередь Queue и словарь посещённых клеток visited
  2. 4 направления: (1,0), (0,1), (-1,0), (0,-1)
  3. Стандартный BFS по сетке
  4. При достижении цели — восстановление полного пути
     от конца к началу через словарь visited
  5. Возвращает список всех клеток от старта до цели
  6. Если путь не найден — возвращает null

Используется для:
  - Генерации маршрутов патруля
  - Преследования игрока
  - Проверки достижимости клеток


============================================================
МАРШРУТ ПАТРУЛЯ
============================================================

Метод: GeneratePatrolPath(startX, startY)

  1. Сбор всех проходимых клеток карты
  2. Выбор 3 случайных достижимых точек
  3. Для каждой точки — поиск пути через BFS
  4. Добавление всех клеток пути в общий маршрут
  5. Добавление возврата к стартовой точке
  6. Если маршрут пустой — запасной вариант из 2 клеток


============================================================
ИИ ВРАГОВ
============================================================

Метод: MoveEnemyStep(enemy) — вызывается рекурсивно для бесконечного движения

Логика обнаружения игрока:
  - Вычисление дистанции до игрока: sqrt(dx^2 + dy^2)
  - Если дистанция < 5 клеток:
      -> IsChasing = true
      -> оповещение соседних врагов в радиусе 8 клеток
  - Если дистанция > 7 клеток:
      -> IsChasing = false (потерял игрока из виду)

В режиме преследования (IsChasing == true):
  1. BFS-поиск пути к игроку: FindFullPath(враг, игрок)
  2. Если путь найден: target = path[0] (первый шаг к игроку)
  3. Если путь не найден: сброс в патруль

В режиме патруля (IsChasing == false):
  1. PatrolIndex = (PatrolIndex + 1) % PatrolPath.Count
  2. target = PatrolPath[PatrolIndex]

Движение к цели:
  1. Вычисление fromLeft/fromTop и toLeft/toTop
  2. Вычисление расстояния: sqrt((toX-fromX)^2 + (toY-fromY)^2)
  3. Выбор скорости: ChaseSpeed (погоня) или BaseSpeed (патруль)
  4. Длительность анимации: max(0.08, (distance/CellSize) / speed)
  5. Два DoubleAnimation на Canvas.Left и Canvas.Top
  6. FillBehavior = HoldEnd
  7. По завершении — рекурсивный вызов MoveEnemyStep

Кооперация врагов (AlertNearbyEnemies):
  - Когда один враг замечает игрока, он оповещает всех
    соседних врагов в радиусе 8 клеток
  - Соседи тоже переходят в режим преследования


============================================================
ФОРМУЛА СКОРОСТИ ВРАГОВ
============================================================

Генерация скорости (CreateEnemySprite):
  BaseSpeed  = (0.3 + Random(0.0..0.2)) * difficultyMultiplier
  ChaseSpeed = BaseSpeed * 1.6

Пример для Normal (multiplier = 1.0):
  BaseSpeed  = 0.3 .. 0.5
  ChaseSpeed = 0.48 .. 0.8

Формула длительности анимации:
  duration = (distance / CellSize) / speed
  где distance — расстояние в пикселях
        CellSize = 40
        speed — BaseSpeed или ChaseSpeed

  При distance = 40 (1 клетка), speed = 0.5:
    duration = (40/40) / 0.5 = 1 / 0.5 = 2.0 сек

  При distance = 40, speed = 5.0 (Ад):
    duration = (40/40) / 5.0 = 1 / 5.0 = 0.2 сек

  speed в знаменателе -> больше speed = меньше duration = быстрее враг


============================================================
СЛОЖНОСТЬ
============================================================

Выбирается в главном меню один раз на сессию.
По умолчанию: Нормал.

| Сложность | multiplier | Врагов | Сокровищ | Музыка       |
|-----------|-----------|--------|----------|--------------|
| Легкая    | 0.7       | 3      | 3        | level_music  |
| Нормал    | 1.0       | 4      | 5        | level_music  |
| Хард      | 1.5       | 4      | 8        | level_music  |
| Ад        | 10.0      | 5      | 10       | hell_music   |

Применение множителя:
  - Скейлит ТОЛЬКО скорость врагов (BaseSpeed и ChaseSpeed)
  - Не влияет на паттерн движения, радиусы обнаружения, BFS
  - Адский режим использует отдельную музыку (hell_music.mp3)
  - В HUD отображается метка "АД" для адского режима

Все значения можно скейлить в разные стороны, для достижения идеального
баланса врагов.


============================================================
СОКРОВИЩА
============================================================

Спавн (SpawnTreasures):
  1. Сбор всех проходимых клеток (кроме старта игрока)
  2. Случайный выбор treasureCount позиций
  3. Создание Rectangle 20x20 с текстурой treasure.png
  4. Анимация мерцания:
     - DoubleAnimation на Opacity
     - From = 1.0, To = 0.4
     - Duration = 0.5 сек
     - AutoReverse = true
     - RepeatBehavior = Forever

Сбор (CheckTreasurePickup):
  1. Сравнение координат игрока и каждого сокровища
  2. Остановка анимации мерцания: BeginAnimation(Opacity, null)
  3. Анимация исчезновения:
     - DoubleAnimation на Opacity
     - From = 1.0, To = 0.0
     - Duration = 0.2 сек
     - FillBehavior = Stop
  4. Удаление с холста и из списка
  5. score++
  6. Если score >= totalTreasures — победа


============================================================
СТОЛКНОВЕНИЯ И УРОН
============================================================

Таймер (DispatcherTimer):
  - Интервал: 50 мс (~20 проверок/сек)
  - Метод: CollisionCheck

Проверка коллизий:
  1. Если !isGameRunning или isInvulnerable — выход
  2. Получение границ игрока (Rect)
  3. Для каждого живого врага:
     - Получение границ врага (Rect)
     - Расширение хитбокса врага на 4px (Inflate)
     - Проверка пересечения: IntersectsWith
  4. При пересечении — HitByEnemy()

Обработка урона (HitByEnemy):
  1. lives--
  2. Если lives <= 0 — GameOver()
  3. Иначе:
     - isInvulnerable = true
     - Остановка анимаций движения игрока
     - Анимация мигания (неуязвимость):
       Opacity 1.0 <-> 0.2, 7 циклов, AutoReverse = true
       Duration = 0.12 сек
     - Анимация отбрасывания на старт (1,1):
       Canvas.Left/Top -> позиция (1,1)
       Duration = 0.4 сек, FillBehavior = HoldEnd
     - По завершении мигания: isInvulnerable = false


============================================================
ПОБЕДА
============================================================

Условие: score >= totalTreasures

WinLevel():
  1. isGameRunning = false
  2. Остановка таймера коллизий
  3. Анимация победной пульсации игрока:
     - Width: 36 -> 52, Duration = 0.4 сек
     - Height: 36 -> 52, Duration = 0.4 сек
     - AutoReverse = true
     - RepeatBehavior = 4 цикла
  4. Сообщение: "УРОВЕНЬ ПРОЙДЕН! R - ЗАНОВО, ESC - МЕНЮ"


============================================================
ПОРАЖЕНИЕ
============================================================

Условие: lives <= 0

GameOver():
  1. isGameRunning = false
  2. Остановка таймера коллизий
  3. Анимация затемнения игрока:
     - Opacity: 1.0 -> 0.3
     - Duration = 0.8 сек
     - FillBehavior = HoldEnd
  4. Показ меню смерти:
     - Фон: death_bg.png
     - Заголовок: "ВЫ ПОГИБЛИ!"
     - Кнопка "ПЕРЕЗАПУСТИТЬ УРОВЕНЬ"
     - Кнопка "ГЛАВНОЕ МЕНЮ"
     - Музыка: death_music.mp3 (однократно)
     - Бинды: R = рестарт, Esc = главное меню


============================================================
АНИМАЦИИ WPF (свойства зависимостей)
============================================================

Все анимации в проекте используют DoubleAnimation + BeginAnimation:

1. Движение игрока:
   Свойства: Canvas.Left, Canvas.Top
   FillBehavior: HoldEnd

2. Движение врагов (патруль и преследование):
   Свойства: Canvas.Left, Canvas.Top
   FillBehavior: HoldEnd
   Динамическая длительность: зависит от расстояния и скорости

3. Отбрасывание игрока при уроне:
   Свойства: Canvas.Left, Canvas.Top
   FillBehavior: HoldEnd
   Duration: 0.4 сек

4. Мерцание сокровищ:
   Свойство: Opacity
   From: 1.0, To: 0.4
   AutoReverse: true
   RepeatBehavior: Forever

5. Исчезновение сокровища при сборе:
   Свойство: Opacity
   From: 1.0, To: 0.0
   Duration: 0.2 сек
   FillBehavior: Stop

6. Мигание игрока (неуязвимость):
   Свойство: Opacity
   From: 1.0, To: 0.2
   AutoReverse: true
   RepeatBehavior: 7 циклов
   Duration: 0.12 сек

7. Затемнение игрока при смерти:
   Свойство: Opacity
   From: 1.0, To: 0.3
   Duration: 0.8 сек
   FillBehavior: HoldEnd

8. Победная пульсация игрока:
   Свойства: Width, Height
   From: 36, To: 52
   AutoReverse: true
   RepeatBehavior: 4 цикла
   Duration: 0.4 сек

ТАЙМЕРЫ (DispatcherTimer)
============================================================

1. Таймер коллизий:
   Интервал: 50 мс
   Назначение: проверка пересечений игрока с врагами

2. Таймер задержки спавна врагов:
   Интервал: 200-800 мс (случайный)
   Назначение: враги начинают движение не одновременно

3. Таймер фейк-загрузки:
   Интервал: 0.5 сек (анимация точек)
   Интервал: 3 сек (задержка перед запуском)
